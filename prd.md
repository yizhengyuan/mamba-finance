# 产品需求文档 (PRD)：Mamba Finance

**版本：** V1.0 (Build 2026)
**定位：** 极简主义、AI 驱动的资产与债权管理系统（Web 版）

---

## 0. 品牌寓意 (Brand Identity)
*   **Mamba (曼巴)：** 象征**极致精度**、**反应速度**与**掌控力**。寓意系统能像曼巴一样精准盯紧每一分钱，绝不遗漏任何一个还款节点。
*   **Finance (财务)：** 专注于**资产增值**与**债权管理**，而非简单的流水记账。
*   **核心理念：** “精准出击，账目清晰。”

---

## 1. 项目概述
### 1.1 项目背景
针对典当行及民间借贷业务中，传统记账软件（如 iCost）功能太泛、而 Excel 追踪还款计划太累的痛点，打造一个专门针对“借贷订单周期”的 Web 版财务管理工具。

### 1.2 核心目标
*   **自动化：** 一键生成还款计划，取代人工计算。
*   **可视化：** 建立基于“还款日历”的财务看板。
*   **AI 化：** 引入 AI Agent 自动审计账目、识别凭证并生成每日财报。

---

## 2. 核心功能模块

### 2.1 账户与资产中心 (iCost Style)
*   **多维度账户：** 支持现金、银行卡、微信、支付宝等多个资金池。
*   **资产概览：** 实时展示“总资产”、“待回本金（债权）”、“本月预计利息收入”。
*   **视觉规范：** 继承深色模式 (#121212)，使用卡片式布局。

### 2.2 借贷订单生命周期管理 (The Core)
*   **订单创建：** 录入借款人、本金、月利率、起止日期、抵押物描述。
*   **计划引擎：** 系统根据订单逻辑**自动拆解**为多期还款任务。
    *   *例：* 借 10,000 元，3 个月。自动生成：[3月10日收息100]、[4月10日收息100]、[5月10日收本息10100]。
*   **凭证墙：** 每笔业务均可挂载“转账截图”和“抵押物照片”，支持一键全屏查验。

### 2.3 记账与对账逻辑
*   **收息入账：** 记账时强制关联对应“还款期数”。
*   **状态同步：** 收到款项后，还款计划状态自动从“待收”转为“已收”，并同步更新账户余额。

### 2.4 AI 财务助手 (Mamba AI Agent)
*   **自动简报：** 每天自动扫描到期款项，在首页推送：“今日有 3 位借款人到期，总计应收 1,500 元。”
*   **智能审计：** 监控异常交易（如利息不足额、逾期天数过长），自动标记风险。

## 3. 技术规格 (Technical Specifications)

### 3.1 核心数据结构 (DB Schema)
```typescript
interface Order {
  id: string; // 订单ID
  borrower: string; // 借款人
  principal: number; // 本金
  rate: number; // 月息率
  collateral: string; // 抵押物
  status: 'active' | 'closed' | 'overdue'; // 订单状态
}

interface RepaymentPlan {
  id: string;
  orderId: string; // 关联订单
  dueDate: Date; // 应收日期
  amount: number; // 应收金额
  type: 'interest' | 'principal_and_interest'; // 类型：息/本息
  isPaid: boolean; // 是否已结清
  transactionId?: string; // 关联的具体流水ID
}
```

### 3.2 UI 开发指南
*   **框架：** React/Next.js + Tailwind CSS。
*   **交互：** 减少页面跳转，利用侧边抽屉（Side Drawer）完成 90% 的录入操作。
*   **图表：** 使用简约的折线图展示资产趋势，饼图展示资产构成。

---

## 4. 开发优先级 (Roadmap)
1.  **Phase 1 (MVP)：** 实现账户体系、订单录入、自动还款计划生成、基本收息功能。
2.  **Phase 2 (UX 增强)：** 增加“还款日历”视图、附件图片预览墙、仿 iCost 统计图表。
3.  **Phase 3 (AI 集成)：** 接入 AI 每日总结简报、OCR 转账凭证自动关联。

---

## 5. 验收标准
*   **准确性：** 自动生成的还款计划金额需精确到 0.01 元。
*   **效率：** 用户从登录到查看今日逾期款项的操作路径不超过 2 步。
*   **体验：** UI 整体风格需保持与 Apple iCost 高度一致的极简质感。
