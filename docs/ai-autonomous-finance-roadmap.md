# AI 全自动财务中台愿景（冻结版）

最后更新：2026-03-04  
状态：`ON_HOLD`（当前阶段不实现，后续有时间再深化）

## 1. 背景

当前 Mamba Finance 已完成可用主链路（订单、还款计划、核销、附件、Dashboard）和 Phase 2 体验增强。  
团队确认“AI 全自动财务中台”方向有价值，但短期复杂度过高，决定先冻结实现。

## 2. 愿景定义

“AI 全自动财务中台”指：
- 数据自动进入系统（而非人工逐条录入）
- AI 自动抽取与理解交易信息（金额/时间/对手方/关联订单）
- 自动完成对账与异常发现
- 自动生成经营简报与风险处置建议
- 人工只做关键审批与例外处理

## 3. 当前与目标差距

### 已具备
- 稳定数据模型与事务链路
- 可观测基础（统一错误响应、关键业务日志）
- 日历、图表、附件预览、日常简报基础能力

### 尚未实现
- 自动化数据采集通道（银行/支付平台/API/邮箱）
- OCR/语义抽取流水线（含置信度与可追溯证据）
- 自动对账引擎（匹配、冲突检测、差异闭环）
- AI 风险审计与处置建议引擎
- 调度与自动执行体系（定时任务、重试、告警）
- 企业级治理（权限、审计、监控、备份恢复）

## 4. 分阶段路线（未来恢复时使用）

### Phase 3A：数据采集与抽取（建议先做）
- 接入凭证 OCR（先单来源）
- 建立结构化抽取结果表与审核流
- 附件 -> 交易候选 的半自动关联

### Phase 3B：自动对账与异常检测
- 规则引擎实现“订单-计划-流水-账户”自动对齐
- 增加异常分级（高/中/低）与待办队列

### Phase 3C：AI 运营化
- 生成日报/周报
- 风险建议与动作推荐
- 审批闭环与回溯分析

## 5. 恢复开发前检查清单

恢复该方向前，建议先满足：
- 真实生产数据样本可用于验证（脱敏）
- 明确 OCR/LLM 成本预算与 SLA 目标
- 先完成权限与审计日志基线
- 评估合规要求（数据保留、操作留痕）

## 6. 当前决策记录

- 决策：暂不实现 AI 全自动中台，先保持现有系统稳定迭代。  
- 原因：当前实现复杂度高、投入产出比不理想。  
- 处理：本文件作为项目内长期路线记录，后续直接从本文件恢复规划。
